When to Use Plan Mode

A decision tree for every task. Takes 10 seconds. Saves you 30 minutes.

1
Is it a simple, single-file change? Typo fix, color change, small text edit

YES Skip Plan Mode. Just do it.

↓ NO — Keep going

2
Are you working in a project you didn't build or haven't touched recently?

YES Use Plan Mode. Explore first.

↓ NO — Keep going

3
Will this task create new files or change your project structure?

YES Use Plan Mode. Plan the changes.

↓ NO — Keep going

4
Could Claude misunderstand what you want? Vague request, multiple approaches, hard to explain

YES Use Plan Mode. Let it ask Qs.

↓ NO — Keep going

5
Will this touch 3 or more files?

YES Use Plan Mode.

↓ Probably fine without it

?
Still unsure? If any voice in your head says "I wonder what Claude's going to do with this" — that's your signal. Use Plan Mode.

The 4-Phase Workflow

1Explore Plan Mode. "Read this project and tell me what you see."
2Plan Plan Mode. "I want to add X. What's the approach?"
3Implement Normal Mode. "Execute the plan. Start with step one."
4Commit Normal Mode. "Commit this progress."

Keyboard Shortcuts

Shift+Tab Toggle Plan Mode on/off
Ctrl+G Open plan in your text editor
Esc x2 Cancel current action
/clear Reset conversation (files stay)

The Restaurant Rule

You wouldn't let a chef bring you random plates without reading the menu first. Don't let Claude Code start building without showing you the plan. Read the menu. Decide what you want. Then order.
