Project Workflow Checklist

Follow this for every project. Same workflow. Gets faster every time.

1
PLAN
Plan Mode
  • Switch to Plan Mode (Shift+Tab)
  • Describe what you want to build in plain English
  • If existing project: "Read this project and tell me what you see"
  • Ask Claude to propose an approach with numbered steps
  • Review the plan. Edit if needed (Ctrl+G)
Don't skip this. Five minutes of planning saves thirty minutes of undoing.
2
CONFIGURE
Plan Mode
  • Create or update CLAUDE.md for this project
  • Include: project overview, communication style, rules
  • Add business context (audience, offers, tone)
  • Create CLAUDE.local.md for private info (API keys, client names)
3
BUILD
Normal Mode
  • Switch to Normal Mode (Shift+Tab)
  • Reference the plan: "Execute the plan. Start with step one."
  • Review each change as it happens
  • Use subagents for independent parallel tasks
  • Commit progress after each major milestone
The loop: Prompt > Review > Refine > Repeat. Don't just let it run.
4
TEST
Normal Mode
  • Ask Claude to create test data if needed
  • Run it and check the output yourself
  • Try to break it (bad input, edge cases)
  • Be specific about bugs: what happened vs. expected
  • Re-test after each fix
Actually test. Open it in a browser. Run the script. Click the buttons.
5
SHIP
Normal Mode
  • Final review of all changes
  • Commit with descriptive message
  • Deploy if applicable
  • Update project notes with what was completed

Claude's responses feel off or confused
You've pivoted from original direction
Conversation is very long
Starting a new phase of the project

Your files, CLAUDE.md, and git history are all preserved. /clear only resets the conversation.
